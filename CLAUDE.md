# AI 영상학습 사용자 포털 — 프로젝트 가이드

한국지역정보개발원(KLID) / 행정안전부 수탁사업 "AI 기반 지방정부 CCTV 관제지원시스템 구축(2차)" 중
**외부 기업·연구자 대상 AI 영상학습 사용자 포털**의 프런트엔드 목업 프로젝트.

- 현재 단계: **1차 목업** (구조가 보이는 수준 → 7월 초 기업 간담회용)
- 일정: 6월 말 1차 목업 / 7월 말 상세 목업·룩앤필 / 8월 개발 착수
- 성격: 내부 관제용 CUVIA 대시보드와 **별개 아이덴티티**. 외부 일반 사용자 대상 공공 포털.

> **기획 문서는 [docs/](docs/) 폴더.** 기획 정본 [docs/02_기획/작성/기획정리_최종.md](docs/02_기획/작성/기획정리_최종.md), 회의록 [docs/00_회의록/260611.md](docs/00_회의록/260611.md), 기획 수신 원본 `docs/02_기획/수신/`, 원본자료(RFP 등) `docs/99_source/`. 노출 문구·수치는 RFP와 회의록 근거만 사용.

- **Vite + React + TypeScript**

- 공통 컴포넌트: **직접 최소 구축** (외부 UI 라이브러리 미사용)

## 디자인 방향

- **라이트 톤 신규**. 밝은 흰/회색 배경 + 브랜드 포인트 컬러 + 넓은 여백. 공공기관 포털 표준 룩.
- 다크 테마 없음(목업 범위 제외).

### 타이포그래피

- 폰트: **Pretendard**
- 본문 base **15px** (공공포털 표준)
- **최소 폰트 13px. 12px 이하 사용 금지.** 캡션·메타도 13px 하한.
  토큰에서 `text-xs`를 13px(0.8125rem)로 재정의해 하한을 강제했다. 즉 어떤 클래스를 써도 13px 미만은 안 나온다.
- 제목 스케일은 토큰으로 관리(`text-h1`~`text-h4` 등), 임의 px 박지 말 것.

### 컬러

- 중립 그레이 스케일 + Primary(브랜드 1색) + 시맨틱 4색(success/warning/danger/info)
- **이벤트 유형색**(산불/침수/화재/유괴 등)은 토큰으로 정의해 검색카드·배지에서 일관 사용
- **opacity 색 배경 토큰 금지**: `bg-red-500/10` 류 안 씀. 배경은 표면 토큰, 색은 보더·텍스트·아이콘에만.

### 레이아웃

- 글로벌: **상단 GNB + 푸터**(포털형). 관제 대시보드의 사이드바 구조 아님.
- 레이아웃 분기: 메인(풀폭) / 일반 콘텐츠(컨테이너) / 워크스페이스 / 관리자(사이드바형).
- z-index는 레이어 토큰으로 관리. **지도 위 요소(마커·팝업)는 항상 패널·버튼보다 아래**, MapContainer에 `isolation: isolate`.

## 코드 컨벤션 (계승 룰)

- **한국어 노출 문구에 em-dash(—) 금지.** toast/hint/description/label/dialog 등은 마침표·쉼표·가운뎃점·콜론으로 대체.
- **날짜·시각은 `@/lib/datetime` 공통 유틸**(`formatDate`/`formatDateTime`)만 사용. `toLocaleTimeString("ko-KR")` 금지(오전/오후 붙음).
- **Modal vs Dialog 명명**: Modal = 폼/picker 큰 창(`XxxModal`), Dialog = 결정 확인 짧은 팝업(`XxxDialog`).
- **모달 푸터에 "닫기" 버튼 금지**(우상단 X 기본 제공). "취소"·"머무르기" 등 의미 액션은 예외.
- **글자수 카운터**: Textarea는 내부 우하단 showCount, Input은 아래 줄 `justify-between` 좌측 에러·우측 카운터(13px gray-500 mono).
- 작업 범위는 **이 폴더(`klid_2nd`)만**. 형제 폴더로 확장 금지.

## 폴더 구조

```
src/
  app/         App.tsx 라우터
  krds/        krds-tokens.css (KRDS 디자인시스템 토큰·폰트. components/krds + Storybook 전용)
  lib/         cn, datetime, format (범용 유틸)
  mocks/       klid.ts (KLID 기관정보 샘플 데이터)
  components/
    krds/      ★ 본작업 정본 디자인 시스템. Button, TextField, Table, Card … + KRDS Header/Footer

  mockup/      목업. 메인·플로우 확인용 (목업 관련 코드·데이터는 전부 이 안에서 관리)
    styles/         index.css (Tailwind 토큰 — mockup 전역 스타일)
    domain/         eventTypes·dataset 도메인 상수 (이벤트 유형색·데이터셋 아이콘)
    pages/
      main/           메인·홈 (구 landing-v2, 원본은 docs/98_아카이브/mockup_hero)
      PlaceholderPage
      StyleGuide       디자인 스타일 가이드 (/style)
    components/
      layout/         AppLayout·AppHeader·AppFooter (코발트 셸)
      ui/             Container·Button(캡슐)·Reveal·Mark·CountUp
      Logo
    mocks/            landing.ts (메인·포털 공통 샘플 데이터)
```

- **본작업은 `components/krds`를 디자인 수정해서 페이지를 만든다.**
- **목업 작업은 `mockup/` 안에서 한다.** 메인·플로우로 서비스 정체성 + 디자인 스타일 확인이 목적. 디테일 기능·본격 디자인은 추후.
- `mocks/` 는 화면 채우는 샘플 데이터. klid.ts(기관정보), mockup/mocks/landing.ts(메인·포털 공통).

## 포털 기능 범위 (회의 1차 정리본 기준)

핵심 흐름: **데이터 검색·반출 / 데이터 제작(저작도구·생성형 AI) / 워크스페이스**

| 화면 | 내용 |
|---|---|
| 메인 | 데이터 통계, 지역별 지도 현황, 기능 소개, 공지사항 |
| 데이터 검색 | 검색 카드(이벤트 유형 중심) + 조건 필터(이벤트·지역 핵심, 기간·시간대·키워드 보조) + 결과 목록 |
| 검색 결과 상세 | 영상/이미지 미리보기, 데이터 상세 |
| 반출 요청 | 요청 요약(이벤트·기간·영상수·이미지수), 처리 상태, 다운로드 기한 |
| 워크스페이스 | 반출 현황, 처리 상태, 다운로드 관리, 유효기간, 생성형 AI·저작도구 작업 현황 |
| 저작 도구 | 영상 업로드 → 이미지 컷 → 수동 라벨링 → 증강 (오토 라벨링·VLM 제외) |
| 생성형 AI | 이미지 업로드 → 영상 생성 → 다운로드 (저작도구 자동 연동 제외) |
| 마이페이지 | 회원 정보·계정 설정 (워크스페이스와 역할 분리) |
| 관리자 | 데이터/사용자/삭제/검색 카드 관리 |

### 올해 범위 제약 (목업 반영 시 주의)

- 검색은 **조건 검색** 중심(AI 자연어 검색 제외)
- 데이터 제공 단위 = **영상+이미지 세트**(영상 포함/제외 옵션은 검토 중)
- 계정 = **1유저 1워크스페이스**(기업 계정·하위 사용자·공유·권한 세분화는 내년)
- 개인정보 비식별·암호화는 올해 제외 → "개인정보 포함 데이터 업로드 금지" 안내 + 관리자 삭제 수준
- 업로드물·생성물·반출 데이터셋은 **유효기간 명시 후 만료 시 삭제** → 목록에 다운로드 기한 표시 필수
