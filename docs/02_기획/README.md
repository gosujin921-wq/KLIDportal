# 02_기획 폴더 안내

기획 단계 자료를 **출처 기준**으로 나눈다. (기획에서 받은 원본 ↔ 우리가 작성한 산출물)

## 수신/ — 기획에서 받은 원본
폴더명 = `날짜_v버전`(수신일 + 개정 버전). 옛 "1차/2차"는 버전 개념이라 헷갈려 이 형식으로 대체.

- `수신/2026-07-09_v1/` (구 목업_1차): design-request 2종 + 저작도구 레퍼런스 png 4장
- `수신/2026-07-10_v2/` (구 목업_기획2차, 개정): design-request 2종(메인 산출물 5장→4장) + 저작도구 png 7장

> 수신 원본은 그대로 보존한다(내용 편집 금지). 두 버전 모두 내부 헤더 작성일은 2026-07-03이라, 구분은 폴더명(수신일·버전)이 맡는다.

## 작성/ — 우리가 작성한 산출물 (2개)
- **`기획정리_최종.md`** ★ 단일 기획 정본: 기획 v2 요구사항 + 검색·반출 기준 + 열린 결정(§13) + 결정 로그(§14). 구 `기획요구사항_정리.md`·`기획보강_데이터셋검색기준.md`를 통합
- `화면정의서_1차목업.md`: 1차 목업 작업 기록(레이아웃·기능·사용성). 본문은 v1 기준, **§0-A v2 델타가 본문을 덮어씀**

> 파일명의 "1차"는 design-request 버전이 아니라 **1차 목업(프로젝트 단계)**을 뜻함.
