**정합 이슈 2건 (결정 필요, 정의서는 양쪽 병기)**

1. **GNB 구성이 코드와 기획 문서가 다름.**
   - 현행 코드: `데이터 검색 | 저작 도구 | 생성형 AI | 워크스페이스` (제작 기능이 GNB 최상위)

2. **이벤트 유형 수.** 코드(`eventTypes.ts`)는 5종(산불·침수·화재·유괴·기타), 요청서는 8종(쓰러짐·폭력·화재·교통사고·유괴·침수·산불·산사태). 8종 채택 시 토큰(`--color-event-*`)·아이콘 4종 추가 필요. 유형 세트 확정 필요.

### 0-0-1. 디자인 스타일 어휘 (index.css 확정 토큰, 전 화면 공통)

디자인 md 작업 시 임의 색·크기 대신 아래 어휘만 사용한다.

- **Primary**: cobalt-600(#2e45dc)이 메인 강조. 라이트 베이스(흰 배경 + slate 그레이).
- **타이포**: Pretendard, 본문 15px, **최소 13px(12px 이하 금지)**. 히어로 H1 52px.
- **시맨틱 4색**: success #16a34a / warning #d97706 / danger #dc2626 / info(=cobalt) #2e45dc.
- **이벤트 유형색**: `--color-event-*` 토큰. 배지는 "옅은 불투명 배경(orange-50류) + 유형색 텍스트·보더" 패턴(opacity 배경 토큰 금지 룰).
- **재사용 가능한 비주얼 유틸**: `.hero-gradient`(히어로 아이시 블루), `.band-cobalt`+`.band-dots`(딥 코발트 밴드. Wow·CTA·관리자 사이드바 후보), `.card-glow`(카드 호버 글로우), `.text-gradient-cobalt`(빅 넘버 강조), CountUp·Reveal·Mark(모션 유틸).
- 서브페이지는 랜딩만큼 화려할 필요 없음. **히어로·밴드 유틸은 랜딩·소개 페이지 한정**, 서브페이지는 흰 배경 + 카드 + cobalt 포인트로 절제.

이 목업의 목적은 **디테일 기능이 아니라 (1) 전체 플로우로 "우리 서비스가 무엇인지" 보여주고 (2) 디자인 스타일을 확인받는 것**이다. 모든 화면 정의는 이 목적에 종속된다.

| 원칙 | 내용 |
|------|------|
| 이동은 진짜, 데이터는 가짜 | 페이지 간 라우팅·클릭 이동은 실제로 동작. 콘텐츠는 mock 고정 데이터 |

## 1. 공통 레이아웃

### 1-1. 헤더 + GNB

```
┌────────────────────────────────────────────────────────────────────┐
│ [로고] AI 영상학습 사용자 포털   학습데이터 워크스페이스 소식&참여 이용안내 │
│                                              ≡ 사이트맵 │ 로그인 [회원가입] │
└────────────────────────────────────────────────────────────────────┘
```

- **현행 AppHeader 재사용**: sticky + 백드롭 블러, 활성 메뉴 cobalt-700, 우측 로그인(ghost)·회원가입(primary) 구성은 이미 구현돼 있음. NAV 배열만 확정 IA로 교체(§0-0 이슈 1).
- 로그인 시 우측: `로그아웃 · 마이페이지`. 데모 토글은 우측 끝에 개발용으로만 배치(시연 시 숨김 가능).
- GNB 현재 섹션: 하단 인디케이터(밑줄) + 볼드. 호버 시 하위 메뉴 드롭다운(워크스페이스·소식&참여·이용안내만, 학습데이터는 바로 이동).
- 워크스페이스 비로그인 클릭: 로그인 유도 다이얼로그 또는 잠금 화면으로 이동(11번 화면).
- **사용성**: GNB 4개는 텍스트만으로 성격이 갈리도록 순서 고정(찾기 → 만들기 → 소식 → 안내). 스크롤 시 헤더 고정(sticky)으로 어느 화면에서도 전환 가능.

### 1-2. 푸터

요청서 그대로 3단(기관 로고·소셜 / 주소·연락처 / 약관·방침·사이트맵·패밀리사이트). 약관·방침은 이용안내 해당 탭으로 연결.

### 1-3. 페이지 헤더 블록 (서브페이지 공통)

```
브레드크럼: 홈 > 소식&참여 > 공지사항        ← 현재 위치 비링크
페이지 타이틀 (H1) + 한 줄 설명(선택)
[LNB: 좌측 사이드바 or 상단 탭]
```

- 레이아웃 유형은 요청서의 A(풀폭)/B(좌 LNB)/C(상단 탭)/D(관리자)를 그대로 채택.
- **사용성**: 유형 B·C 모두 활성 메뉴는 "색+굵기+인디케이터" 3중 표시(색맹 대비). 브레드크럼의 GNB 단계는 해당 섹션 기본 진입 페이지로 링크.

| 표시 | 아코디언(FAQ·약관), 진행바, 툴팁, 토스트, 빈 상태(일러스트+CTA) | 소식·워크스페이스 |
| 오버레이 | Modal(다운로드 신청 등 폼형), Dialog(로그인 유도 등 확인형) ※ 명명 규칙 준수, 모달 푸터 "닫기" 금지 | 상세·워크스페이스 |
| 시각화 | 막대/도넛/라인 차트(Recharts, 데이터 현황) | 데이터 현황 |

**상태 뱃지 어휘 통일(전 화면)**: 대기·심사중=중립/info, 진행중·처리중=info, 완료·승인=success, 기한임박·검수대기=warning, 실패·반려·만료=danger.

## 12. 디자인 md 작업 순서 제안

랜딩·셸·토큰·스타일가이드는 이미 있으므로, placeholder 4개 라우트를 채우는 것부터 시작한다.

1. **IA 확정 + 라우트 재편**: §0-0 이슈 1(GNB)·이슈 2(유형 세트) 결정 → App.tsx 라우트·AppHeader NAV 교체, 레이아웃 B(LNB 사이드바)·C(상단 탭) 프레임 신설
2. **검색·목록 `/search` → 상세 → 신청 모달**: 데모 핵심 동선. 데이터셋 카드·필터 칩·Modal 정본이 여기서 확립됨
3. **메인 갭 보완**: 히어로 검색 진입점 + (결정 시) 유형 쇼케이스·인기 데이터셋 섹션. 2에서 만든 데이터셋 카드 재사용
4. **워크스페이스 `/workspace`**: 대시보드(상태 뱃지·작업 행 정본) → 업로드·저작도구(`/authoring` 이관)·내 학습데이터·증강·생성형 AI(`/genai` 이관) → 비로그인 잠금
5. **로그인·회원가입**: 폼 스타일 정본(krds TextField 조합) + 데모 로그인 토글
6. **소식&참여**: 데이터 현황(Recharts + RegionSection 지도 재사용) → 공지·FAQ → 문의·활용사례
7. **마이페이지 → 이용안내 → 사이트맵**
8. **관리자**: 대시보드 + 공지 관리, 나머지 골격 (`band-cobalt` 사이드바로 아이덴티티 분리 검토)

각 단계에서 새로 확정된 컴포넌트는 `/style` 스타일가이드에 즉시 추가해 디자인 확인 창구를 유지한다.
