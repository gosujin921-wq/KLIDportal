## 5. 학습데이터 (검색·반출)

### 5-1. 화면 흐름
- GNB 클릭 → 검색·목록 바로 진입(LNB 없음). 검색·목록 → 상세 → 다운로드 신청(모달)
- 상세 브레드크럼: 홈 > 학습데이터 > 데이터셋명
- **검색 모델 = 데이터셋 단위 카탈로그**(v2 확정): "포털 보유 데이터셋을 탐색 → 비교·선별 → 상세 확인 → 다운로드 신청"(DFEAT-001~006). 회의록의 "영상 단위 검색·반출 시 묶음 생성"은 반출 서버 내부 로직으로 흡수 해석(§13 O1)
- **외부 플랫폼 연계 데이터셋**(DFEAT-004)도 동일 검색 경로에 통합 표시

### 5-2. 검색 조건 (회의록 §5 + REQ-006 + v2)

| 조건 | 중요도 | 비고 |
|------|--------|------|
| **이벤트 유형** | 핵심 | 8종 |
| **지역** | 핵심 | 17개 시·도 |
| 시간대 | 보조 | 주간/야간/새벽 |
| 기간 | 보조 | 기본값 전체 기간 |
| 키워드 | 보조 | |
| 태그 | 보조 | v2 추가 |
| 출처 | 보조 | **원본/수동라벨링/AI증강** (DFEAT-001). 시스템 제공 데이터셋 대상, 워크스페이스 저작 데이터는 검색 대상 아님. ※목업 미반영 |

- 화면에서 유형·지역을 가장 눈에 띄게, 나머지는 하위 배치
- 결과 목록: 정렬·페이지네이션 + 즐겨찾기 토글 + **AI 증강 배지**(시스템 제공 증강 데이터셋 표시, DFEAT-002. ※목업 미반영). 상세: 메타·썸네일 미리보기·파일 구성(DFEAT-003)
- 결과 0건: 안내 + 조건 완화·재검색 유도 (DFEAT-001 수용기준)

### 5-3. 검색 카드 (공용 조건 프리셋, 회의록 §7)
- 검색 카드 = 자주 쓰는 **검색 조건을 미리 구성한 프리셋** (데이터 묶음 아님). 예: "산불 데이터" = 이벤트 산불/지역 전체/기간 전체
- 카드 선택 → 조건 검색 실행 → 이후 조건 조정. 이벤트 유형 중심 제한된 개수(상한은 O3)
- 관리자 "검색 카드 관리"에서 등록·관리 (회의록 §16)
- 현 목업의 히어로 유형 칩 + 쇼케이스 카드가 이 역할

### 5-4. 검색 조건 즐겨찾기 (사용자 개인 프리셋, 자체 제안·목업 반영)
- **LogiCraft·기획 v2에 없음 확인(7/10, DFEAT 55종 전수)**. 가장 유사한 DFEAT-005는 "데이터(결과물) 즐겨찾기"로 별개 기능. 기획 공식 반영하려면 역제안 필요
- 저장 대상 = 데이터셋(결과물)이 아니라 **검색 조건(query) 자체**. 결과 건수·목록은 저장 시점 고정이 아니라 **조회 시점 실시간 계산**
- 역할 분담(확정): **퀵선택("내 검색조건" 칩)+저장 = 검색 화면** / **관리(이름변경·삭제) = 워크스페이스 > 즐겨찾기 > 검색 조건 탭** (v2 채택에 따라 마이페이지→워크스페이스로 이동)
- 데이터 모델(개념): `{ id, name, conditions{ type, region, time, kind, … }, createdAt }`. 스키마·개수 제한·알림은 백엔드 협의(§12)

### 5-5. 반출(다운로드 신청) 프로세스 (회의록 §8 + REQ-007/009)

```
검색·상세 확인 → 다운로드 신청(모달) → 서버가 영상·이미지·메타를 데이터셋 파일로 구성
  → 처리 상태(생성 중 → 완료 → 다운로드 가능) → 기한 내 다운로드
```

- 신청 시 제공 정보: 이벤트 유형/기간/영상 수/이미지 수/데이터셋 개요/처리 상태/다운로드 기한
- **활용 목적 입력 + 약관 동의** 필수 (REQ-007: 학술 연구/기업 R&D/공공 프로젝트/기타)
- **대용량 ZIP 압축·분할·이어받기** (REQ-009)
- 처리 상태는 워크스페이스 대시보드에서 확인. 제공 단위 = **영상+이미지 세트**(영상 제외 옵션은 O2)

### 5-6. 보관·유효기간 정책 (회의록 §11)
- 반출 결과물·업로드 영상·저작 작업물·증강/생성 결과물은 **유효기간 명시 후 만료 시 삭제**(보존기간 경과 후 파기)
- 다운로드 기한·유효기간은 **목록 화면에 명확히 표시** (D-day 병기 권장)
- 용어 통일(제안, O5): 다운로드 신청 결과="다운로드 기한" / 업로드·저작·생성 보관="보관 기한" / 정책 문구="보존기간 경과 후 파기"

## 14. 결정 로그

- 이벤트 유형 8종·시간대(주간/야간/새벽) 검색 반영. 활용목적 값(학술 연구/기업 R&D/공공 프로젝트), ZIP 분할·이어받기 안내, 일일 사용량 제한, 저작도구 기존 데이터셋 진입점 목업 반영.
- 검색 조건 즐겨찾기: 저장 대상은 조건(query), 결과 건수는 조회 시점 실시간. 퀵선택=검색 화면, 관리=마이페이지(목업 반영).
- 마이페이지 슬림화(목업): 대시보드 제거 → 즐겨찾기·이용 내역·회원 정보 3메뉴, 즐겨찾기 2탭.
- 기획 v2 취합(7/10): 검색 모델 카탈로그형 잠정 해소(O1). 즐겨찾기 배치는 v2와 충돌 → O8 재오픈. 목업 수정 대상은 §0 표의 "목업 영향" 열.
- 폴더 정리: 수신 원본은 `../수신/날짜_v버전/`, 작성물은 `../작성/`.
- **v2 목업 반영 완료(7/10)**: D1 랜딩 지도 제거, D2 워크스페이스(증강+생성형 통합·즐겨찾기 신설, DFEAT-005 테이블+해제 컨펌), D3 마이페이지 3메뉴(대시보드 복원), D4 이용안내 2탭+약관·방침 푸터 전용(/terms·/privacy), D5 카카오 인증, D6 팝업 공지(관리자 설정 컬럼+메인 배너 모달·오늘 하루 안 보기), D7 출처 필터·AI 증강 배지, D8 로그아웃 진행중 작업 경고. O8은 v2 채택으로 확정.
