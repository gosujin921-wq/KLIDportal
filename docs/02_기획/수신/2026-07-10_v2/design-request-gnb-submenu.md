# GNB 하위 메뉴 구조 디자인 요청서

> **작성일**: 2026-07-03
> **대상**: AI 영상학습 사용자 포털 v2 — GNB 각 메뉴의 하위 메뉴 구조
> **산출물**: 각 GNB 메뉴별 하위 메뉴 네비게이션 시안 (내부 콘텐츠 영역은 빈 placeholder)

## 0. 전체 화면 요약

### 0-1. 메뉴구성도

포털 전체 화면을 트리로 정리한 것. ★ = 이번 시안 대상.

## 1. 요청 범위

- GNB 클릭 시 진입하는 **각 섹션의 하위 메뉴(LNB) 구조**만 디자인
- 콘텐츠 영역은 **빈 placeholder** (추후 개별 화면 디자인)
- 공통: 헤더(GNB) + 푸터는 메인 페이지 시안 그대로 적용
- 비로그인/로그인 분기가 필요한 메뉴는 명시

## 4. GNB 메뉴별 하위 구조

### 4-1. 학습데이터

> GNB 클릭 시 **검색·목록 페이지로 바로 진입** (별도 LNB 없음)

```
학습데이터
└─ 검색·목록 (기본 진입) ──── 하위 메뉴 없이 단일 페이지 흐름
    └→ 상세 (목록에서 클릭)
        └→ 다운로드 신청 (모달)
```

- LNB 불필요 — 검색→목록→상세 순차 흐름
- 상세 진입 시 브레드크럼: 홈 > 학습데이터 > 데이터셋명

> **이 페이지의 존재 이유** — 포털의 핵심 가치인 "AI 학습데이터 외부 제공"의 진입점. 사용자는 재난안전 유형·지역·기간·태그 등 복합 조건으로 포털 보유 데이터셋을 탐색하고(DFEAT-001), 정렬·페이지네이션된 결과 목록에서 즐겨찾기 토글과 함께 데이터셋을 비교·선별한 뒤(DFEAT-002), 상세 페이지에서 메타정보·썸네일 미리보기·파일 구성을 확인하고(DFEAT-003) 다운로드를 신청(DFEAT-006)하는 **탐색→비교→획득** 전체 흐름의 시작점이다. 외부 플랫폼 연계 데이터셋(DFEAT-004)도 동일 검색 경로에 통합 표시된다.

- **시안 1장**: 검색·목록 페이지 (콘텐츠 비움, 검색바+필터 영역만 placeholder)

### 4-2. 워크스페이스 (로그인 필수)

> 비로그인 시 **LNB 메뉴 구조는 노출하되, 콘텐츠 영역 블러/잠금 + 로그인 유도 오버레이** 표시

```
워크스페이스
├─ 대시보드         ← 기본 진입
├─ 업로드 영상
├─ 저작도구         ※ MF Remote 임베딩 영역
├─ 내 학습데이터
├─ 데이터 증강
└─ 데이터 즐겨찾기
```

| 하위 메뉴               | 비고                                                                                                                                                                             |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **대시보드**      | 워크스페이스 기본 진입. **진행중 작업의 통합 허브**: 업로드 진행 상태, 저작·증강 작업 현황(DFEAT-011), 진행중 다운로드 상태·재시도 관리(DFEAT-008). 사용자가 현재 진행 중인 모든 작업을 한 화면에서 모니터링하고 즉시 조치할 수 있는 능동적 작업 관리 화면. ※ 마이페이지 대시보드(완료 이력 요약)와 역할 구분 |
| **업로드 영상**   | 사용자가 보유한 영상 파일을 포털에 업로드하고 업로드 현황을 관리하는 화면(DFEAT-012). 저작도구(MF)와 독립적으로 포털이 전담하며, 업로드 완료된 영상은 저작도구 탭에서 마킹→라벨링 워크플로우로 연결된다. 업로드 파일은 NAS에 저장, 보존기간 경과 후 파기 |
| **저작도구**      | 관제지원시스템의 저작도구를 Module Federation Remote로 임베딩하는 화면(DFEAT-013, DFEAT-014). 포털 영상 목록을 조회하여 마킹(프레임 추출)→수동 라벨링(폴리곤 편집) 워크플로우를 수행. 저작도구 자체는 외부 모듈이며, 포털은 영상 목록 API·인증 토큰만 제공 |
| **내 학습데이터** | 사용자가 저작도구에서 편집·라벨링한 결과물을 목록으로 조회하고 다운로드하는 화면(DFEAT-015, DFEAT-051). 저작 결과물은 시스템 미수집이므로 사용자 본인만 워크스페이스에서 다운로드 가능. 출처 표식 관리(DFEAT-019)로 데이터 원본 추적 지원 |
| **데이터 증강**   | 외부 AI를 활용해 이미지를 변환·확장(예: 가을 산불→겨울 산불)하는 데이터 증강 실행 화면(DFEAT-016)과 생성형 AI 영상 생성(DFEAT-053). 일일 사용량 제한·기여도 권한(DFEAT-017)이 적용되며, 결과는 사용자 워크스페이스에만 저장(시스템 미수집) |
| **데이터 즐겨찾기** | 검색 결과·상세 페이지에서 토글한 관심 데이터셋을 모아보는 리스트(DFEAT-005). 데이터유형·데이터명·구분·조회수 표시. 즐겨찾기 해제 시 컨펌 모달. 재검색 없이 관심 데이터에 빠르게 접근하고 비교·검토하기 위한 개인 북마크 기능 |

**시안 필요**:

- 로그인 상태: LNB 6개 메뉴 + 빈 콘텐츠 (대시보드 활성)
- 비로그인 상태: LNB 6개 메뉴 노출 + 콘텐츠 영역 블러/잠금 + 로그인 유도 오버레이

### 4-5. 회원 (독립 페이지)

> GNB 외부 — 헤더 "로그인"/"회원가입" 버튼 또는 로그인 유도에서 진입하는 독립 흐름.
> 공통 헤더(GNB) + 푸터 적용. LNB 없음.

#### 회원가입 (스텝 흐름)

```
회원가입
├─ Step 1: 약관·개인정보 수집 동의
├─ Step 2: 카카오 본인인증
├─ Step 3: 회원정보 입력 (아이디·비밀번호·소속 등)
└─ Step 4: 가입 완료
```

> **존재 이유** — 자체 회원제(ID/PW) 기반 포털 접근 통제의 진입점(FEAT-005). 약관 동의→본인인증→정보입력→완료 4단계 순차 흐름으로 개인정보보호법 의무를 준수하면서 1인 1계정을 보장(ADR-004).

| 단계                          | 비고                                                         |
| ----------------------------- | ------------------------------------------------------------ |
| **약관·개인정보 동의** | 개인정보보호법 의무 준수 절차(DFEAT-020). 서비스 이용약관·개인정보 수집·이용 동의를 필수/선택 구분하여 개별 및 전체 동의. 수집 목적·항목·보유기간을 법률에 따라 명시하며, 필수 동의 미완료 시 다음 단계 진행 불가 |
| **카카오 본인인증**     | 외국인 가입 방지 및 1인 1계정 보장을 위한 본인인증(DFEAT-022, ADR-004). 외부 팝업/리다이렉트 방식(EXTSYS-003), 인증 완료 후 자동 복귀 |
| **회원정보 입력**       | 아이디·비밀번호·소속기관·연락처·이메일 등 회원 프로필 생성(DFEAT-021). 아이디 중복확인 + 비밀번호 정책(길이·조합 규칙) 실시간 검증으로 입력 오류 사전 방지 |
| **가입 완료**           | 가입 성공 안내 + "로그인하기" CTA. 회원가입 4단계 흐름의 종착점 |

#### 로그인

```
로그인 (단일 페이지)
├→ 아이디 찾기 (링크)
├→ 비밀번호 찾기 (링크)
└→ 회원가입 (링크)
```

> **존재 이유** — 포털의 로그인 전용 기능(워크스페이스·문의하기·마이페이지 등)에 접근하기 위한 인증 관문(DFEAT-023, FEAT-006). 계정찾기·회원가입으로의 진입점도 겸함.

| 요소                | 비고                                     |
| ------------------- | ---------------------------------------- |
| **로그인 폼** | 아이디·비밀번호 기반 인증 후 세션 생성(DFEAT-023). 인증 실패 시 안내 메시지 표시. 로그아웃 시 진행중 작업(다운로드/저작)이 있으면 경고 후 세션 종료 |
| **부가 링크** | 아이디 찾기(DFEAT-024) · 비밀번호 찾기·재설정(DFEAT-025) · 회원가입(FEAT-005) 진입점 |

#### 아이디 찾기

```
아이디 찾기
├─ Step 1: 카카오 본인인증
└─ Step 2: 아이디 안내 (일부 마스킹 표시)
```

> **존재 이유** — 아이디를 분실한 기존 회원이 카카오 본인인증(DFEAT-022)을 통해 본인 확인 후 가입 시 등록한 아이디를 일부 마스킹된 형태로 안내받는 계정 복구 수단(DFEAT-024).

#### 비밀번호 찾기·재설정

```
비밀번호 찾기
├─ Step 1: 아이디 입력
├─ Step 2: 카카오 본인인증
└─ Step 3: 새 비밀번호 설정 (정책 안내 표시)
```

> **존재 이유** — 비밀번호를 분실한 기존 회원이 아이디 입력 후 카카오 본인인증(DFEAT-022)을 거쳐 새 비밀번호를 설정하는 계정 복구 수단(DFEAT-025). 비밀번호 정책(길이·문자 조합 규칙)을 실시간 안내.

**시안 필요**:

- 회원가입: 스텝 인디케이터 포함 대표 1장 (Step 1 약관동의 활성) + Step 3 정보입력 1장
- 로그인: 1장
- 아이디 찾기 결과 화면: 1장
- 비밀번호 재설정 화면: 1장

### 4-6. 마이페이지 (헤더에서 진입, 로그인 시만)

> GNB가 아닌 **헤더 유틸리티**에서 진입. 별도 LNB 구조 필요.

```
마이페이지
├─ 메인 (대시보드)    ← 기본 진입
├─ 이용 내역 조회
│   ├─ 데이터 다운로드 내역
│   ├─ 데이터 저작 내역
│   └─ AI 사용 내역 (증강) / 기여도 상세
└─ 회원 정보 관리
```

| 하위 메뉴                 | 비고                                                                                                                                                                                     |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **메인 (대시보드)** | **완료된** 이용 이력의 cross-domain 요약 허브(DFEAT-026). 다운로드 완료 건수(DOMAIN-002), 저작 완료 건수(DOMAIN-003), AI 사용량·기여도 요약(DOMAIN-003)을 카드형으로 집계 표시. 각 카드 클릭 → 해당 하위 메뉴 이동. ※ 워크스페이스 대시보드(진행중 작업의 능동 관리)와 역할 구분 |
| **이용 내역 조회**  | **하위 3개 탭**: ① 다운로드 내역 — 완료된 다운로드 이력 열람(DFEAT-009), ② 저작 내역 — 완료된 저작 작업 이력 열람(DFEAT-015), ③ AI 사용 내역 — 증강·생성형AI 호출 이력 + 기여도 누적·등급·현재 권한 조회(DFEAT-018). 모두 '완료 이력' 중심으로 워크스페이스(진행중)와 구분 |
| **회원 정보 관리**  | 비밀번호·연락처·소속기관 등 회원 프로필 조회·수정(DFEAT-027) + 회원 탈퇴 신청(DFEAT-028). 민감 정보 변경 및 진입 시 현재 비밀번호 재확인 필수. 개인정보 삭제 시 이력은 보존 |

**시안 필요**:

- LNB 3개 메뉴 + 빈 콘텐츠 (메인 활성)
- "이용 내역 조회" 선택 시 내부 탭 3개 표시 (LNB 내 depth 2 또는 콘텐츠 영역 상단 탭)

### 4-7. 관리자 (/admin)

> 사용자 포털과 별도 레이아웃. 관리자 전용 GNB 또는 사이드바.

```
관리자
├─ 대시보드 (메인)    ← 기본 진입
├─ 다운로드 현황 관리
├─ 회원 현황 관리
├─ 공지사항 관리
├─ FAQ 관리
├─ 문의 관리
├─ 활용사례 관리
└─ 콘텐츠 관리
```

| 하위 메뉴                    | 비고                                                                              |
| ---------------------------- | --------------------------------------------------------------------------------- |
| **대시보드**           | 관리자 진입 시 첫 화면. 다운로드 현황(DOMAIN-002)·회원 현황(DOMAIN-001)·최근 문의(DOMAIN-004) 등 cross-domain KPI를 카드형으로 집계하고, 각 관리 화면 바로가기를 제공하는 종합 현황 허브(DFEAT-042) |
| **다운로드 현황 관리** | 전체 다운로드 건수·인기 데이터셋 TOP N·유형별·지역별·기간별 통계를 차트로 시각화하는 대시보드(DFEAT-039). 다운로드 요청 상태 관리(DFEAT-008) + 통계 데이터 엑셀(XLSX) 내보내기 지원 |
| **회원 현황 관리**     | 전체·신규·활성 회원 통계 대시보드(DFEAT-040) + 회원 목록 테이블(검색·필터·페이징)(DFEAT-041) + 부적절 계정 삭제 조치(DFEAT-029). 회원 통계 엑셀 내보내기 지원 |
| **공지사항 관리**      | 공지사항 작성·수정·삭제(DFEAT-030). 제목·내용·첨부파일(이미지·PDF) 편집 + 공개 기간 설정 + 중요 공지 상단 고정 플래그 + **팝업 공지 설정**(팝업 여부·이미지 배너 업로드·팝업 노출 기간). 팝업 설정된 공지는 메인 페이지 진입 시 이미지 배너형 모달로 표시(DFEAT-055). 첨부파일·팝업 이미지 형식·용량 제한 검증 |
| **FAQ 관리**           | FAQ 항목 CRUD + 카테고리 관리·정렬 순서 설정(DFEAT-036). 사용자 측 아코디언 UI(DFEAT-033)에 표시될 질문·답변 콘텐츠를 관리 |
| **문의 관리**          | 사용자 문의 접수 목록 조회 + 관리자 답변 작성 + 상태(접수/답변완료) 관리(DFEAT-037). 사용자 측 문의하기(DFEAT-034)의 관리자 대응 기능 |
| **활용사례 관리**      | AI 학습데이터 활용사례 CRUD(DFEAT-038). 사용자 측 활용사례 조회(DFEAT-035)에 표시될 대표이미지·제목·요약·본문(활용 배경·적용 방법·결과)을 관리 |
| **콘텐츠 관리**        | 이용약관·개인정보처리방침을 WYSIWYG 에디터로 편집·배포·개정이력 관리(DFEAT-049). 새 버전 작성 시 시행일(미래 가능)을 설정하면 해당일에 자동 전환. 모든 과거 버전 보존 + 개정이력(버전·시행일·작성자·변경사유) 조회 |

**시안 필요**:

- 관리자 LNB (좌측 사이드바 권장 — 메뉴 8개) + 빈 콘텐츠 (대시보드 활성)
- 관리자 레이아웃은 사용자 포털과 다를 수 있음 (디자이너 제안 가능)
