# 즐겨찾기 기능 시나리오 확정안

> 기존 목업에 즐겨찾기 관련 화면이 없어 시나리오를 확정하고 전달합니다.
> 아래 내용을 기반으로 관련 화면 디자인을 추가해 주세요.

---

## 1. 기능 개요

포털의 즐겨찾기는 **두 종류**이며, 워크스페이스에서 탭으로 통합 관리합니다.

| 구분 | 검색 조건 즐겨찾기 | 데이터셋 즐겨찾기 |
|------|-------------------|-------------------|
| 대상 | 검색 필터 조합 (조건 세트) | 개별 학습데이터셋 |
| 최대 수량 | **5개** | **100개** |
| 저장 위치 | 검색 페이지 | 검색 결과 목록 / 데이터셋 상세 |
| 노출 위치 | 검색 페이지 카드 영역 | 워크스페이스 > 즐겨찾기 > 데이터 탭 |
| 해제 위치 | 워크스페이스에서만 | 워크스페이스 + 데이터셋 상세 양쪽 |

---

## 2. 검색 조건 즐겨찾기

### 2-1. 저장 진입

- **위치**: 검색 수행 후, 검색 버튼 근처에 "이 검색조건 즐겨찾기" 버튼 배치
- **조건**: 로그인 상태에서만 활성. 비로그인 시 클릭하면 로그인 유도 안내

### 2-2. 저장되는 검색 조건 항목

아래 6가지 검색 필터 조합이 하나의 즐겨찾기로 저장됩니다.

| # | 조건 항목 | 비고 |
|---|----------|------|
| 1 | 기간 | 시작일 ~ 종료일 |
| 2 | 데이터 유형 | 재난안전 8대 유형 (쓰러짐, 폭력, 화재, 교통사고, 유괴, 침수, 산불, 산사태) |
| 3 | 시간대 | |
| 4 | 지역 | 수집 지역 |
| 5 | 데이터 구성 | |
| 6 | 출처 | 원본 / 수동라벨링 / AI증강 |

### 2-3. 별칭 입력

- 저장 버튼 클릭 시 **소형 팝업(인라인 모달)** 노출
- 별칭 입력 필드 1개 + 저장/취소 버튼
- **플레이스홀더**: 선택된 데이터 유형을 기반으로 자동 생성 (예: "쓰러짐, 폭력")
- 별칭 미입력 시 플레이스홀더 값이 그대로 별칭으로 저장

### 2-4. 수량 제한

- 최대 **5개**
- 5개 초과 시 저장 차단 + 안내 메시지:
  > "최대 5개입니다. 기존 항목을 삭제 후 저장해주세요."

### 2-5. 검색 페이지 카드 노출

- **위치**: 검색 페이지 검색창 하단, 카드 형태로 나열
- **카드 구성**: 별칭 + 저장된 조건 요약 텍스트
- **카드 클릭 동작**: 저장된 조건이 검색 필터에 자동 세팅 + 즉시 검색 실행
- **저장 직후 동작**: 카드 영역만 부분 리로드 (페이지 전체 새로고침 아님)

### 2-6. 별칭 수정

- **워크스페이스 > 즐겨찾기 > 검색 조건 탭**에서만 가능
- 별칭(이름)만 수정 가능
- 검색 조건 자체 편집은 불가 (삭제 후 재등록 방식)

### 2-7. 삭제

- **워크스페이스 > 즐겨찾기 > 검색 조건 탭**에서만 가능
- 삭제 확인 모달 후 제거
- 검색 페이지 카드 영역에서 즉시 제거

---

## 3. 데이터셋 즐겨찾기

### 3-1. 저장

- **위치 1**: 검색 결과 목록의 각 데이터셋 항목에 북마크 아이콘
- **위치 2**: 데이터셋 상세 페이지에 북마크 아이콘
- 아이콘 클릭 시 즉시 등록 (별도 팝업 없음)
- 비로그인 시 클릭하면 로그인 유도 안내

### 3-2. 수량 제한

- 최대 **100개**
- 100개 초과 시 저장 차단 + 안내 메시지:
  > "최대 100개입니다. 기존 항목을 삭제 후 저장해주세요."

### 3-3. 노출

- **워크스페이스 > 즐겨찾기 > 데이터 탭**에서 목록 형태로 확인
- 목록 항목: 번호, 데이터유형, 데이터명, 구분, 조회수

### 3-4. 클릭 동작

- 데이터셋명 클릭 시 해당 학습데이터셋 상세 페이지로 이동

### 3-5. 해제

- **워크스페이스 > 즐겨찾기 > 데이터 탭**: 해제 버튼 클릭 → 확인 모달 → 제거
- **데이터셋 상세 페이지**: 북마크 아이콘 토글로 해제 (확인 모달 없이 즉시)

---

## 4. 워크스페이스 즐겨찾기 관리 화면

### 4-1. 탭 구조

```
워크스페이스 > 즐겨찾기
┌──────────────┬──────────────┐
│   데이터     │  검색 조건    │  ← 탭 전환
└──────────────┴──────────────┘
```

### 4-2. 데이터 탭

| 기능 | 설명 |
|------|------|
| 목록 조회 | 즐겨찾기 등록된 데이터셋 리스트 (번호, 데이터유형, 데이터명, 구분, 조회수) |
| 상세 이동 | 데이터명 클릭 → 데이터셋 상세 페이지 |
| 해제 | 해제 버튼 → 확인 모달 → 즐겨찾기 제거 |

### 4-3. 검색 조건 탭

| 기능 | 설명 |
|------|------|
| 목록 조회 | 저장된 검색 조건 리스트 (별칭, 조건 요약, 저장일) |
| 별칭 수정 | 인라인 편집 또는 수정 아이콘 클릭 → 별칭만 변경 가능 |
| 삭제 | 삭제 버튼 → 확인 모달 → 제거 (검색 페이지 카드도 즉시 제거) |
| 조건 적용 | 조건 요약 클릭 → 검색 페이지로 이동 + 조건 자동 적용 + 검색 실행 |

## 5. 디자인 반영이 필요한 화면 목록

| # | 화면 | 추가/수정 요소 |
|---|------|---------------|
| 1 | **검색 페이지** | "이 검색조건 즐겨찾기" 버튼, 별칭 입력 소형 팝업, 검색조건 카드 영역 |
| 2 | **검색 결과 목록** | 각 항목 북마크 아이콘 (데이터셋 즐겨찾기) |
| 3 | **데이터셋 상세** | 북마크 아이콘 (토글), 해제 시 즉시 반영 |
| 4 | **워크스페이스 > 즐겨찾기** | 탭 2개 (데이터 / 검색 조건), 각 탭별 리스트 + 관리 기능 |
| 5 | **공통 컴포넌트** | 수량 초과 안내 토스트/모달, 비로그인 유도 안내 |

## 6. 주요 사용자 흐름 (User Flow)

### 흐름 A: 검색 조건 저장 → 재사용

```
검색 페이지 진입
  → 검색 조건 설정 (유형: 화재, 지역: 서울)
  → 검색 실행
  → "이 검색조건 즐겨찾기" 버튼 클릭
  → 별칭 입력 팝업 ("화재, 서울" 플레이스홀더)
  → 별칭 입력 또는 그대로 저장
  → 카드 영역에 새 카드 추가 (부분 리로드)

다음 방문 시:
  → 검색 페이지 진입
  → 카드 영역에서 "화재, 서울" 카드 클릭
  → 조건 자동 세팅 + 검색 결과 갱신
```

### 흐름 B: 데이터셋 즐겨찾기 → 워크스페이스 관리

```
검색 결과 목록에서 관심 데이터셋 북마크 아이콘 클릭
  → 즉시 등록 (토스트 피드백)

워크스페이스 > 즐겨찾기 > 데이터 탭 진입
  → 등록된 데이터셋 목록 확인
  → 데이터명 클릭 → 상세 페이지 이동
  → 또는 해제 버튼 → 확인 모달 → 제거
```
