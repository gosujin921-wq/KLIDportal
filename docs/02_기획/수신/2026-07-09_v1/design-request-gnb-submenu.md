# GNB 하위 메뉴 구조 디자인 요청서

> **작성일**: 2026-07-03
> **대상**: AI 영상학습 사용자 포털 v2 — GNB 각 메뉴의 하위 메뉴 구조
> **산출물**: 각 GNB 메뉴별 하위 메뉴 네비게이션 시안 (내부 콘텐츠 영역은 빈 placeholder)

---

## 0. 전체 화면 요약

### 0-1. 메뉴구성도

포털 전체 화면을 트리로 정리한 것. ★ = 이번 시안 대상.

```
AI 영상학습 사용자 포털
│
├─ ★ 메인 페이지 ·························· 풀 시안 (별도 요청서)
│   ├─ 비로그인 — 모드A(수치 카드) + 모드B(지도) 각 1장
│   └─ 로그인 — 헤더 차이 + 즐겨찾기 활성
│
├─── GNB ──────────────────────────────────────────
│   │
│   ├─ ★ 학습데이터 ························ LNB 없음
│   │   ├─ 검색·목록 (기본 진입)
│   │   ├─ 상세 ···························· 추후
│   │   └─ 다운로드 신청 (모달) ············ 추후
│   │
│   ├─ ★ 워크스페이스 (로그인 필수) ········ 좌측 사이드바
│   │   ├─ 대시보드 (기본 진입)
│   │   ├─ 업로드 영상

│   │   ├─ 내 학습데이터
│   │   ├─ 데이터 증강
│   │   ├─ 생성형 AI
│   │   └─ [비로그인] LNB 노출 + 콘텐츠 블러/잠금 + 로그인 유도
│   │
│   ├─ ★ 소식&참여 ························ 상단 탭
│   │   ├─ 데이터 현황 (기본 진입)
│   │   ├─ 공지사항 (목록 / 상세)
│   │   ├─ FAQ (카테고리별 아코디언)
│   │   ├─ 문의하기 (작성 / 내 문의 목록 / 상세) ※로그인
│   │   └─ 활용사례 (목록 / 상세)
│   │
│   └─ ★ 이용안내 ························· 상단 탭
│       ├─ 서비스 소개 (기본 진입)
│       ├─ 이용 가이드
│       ├─ 이용약관
│       └─ 개인정보처리방침
│
├─── 독립 페이지 ──────────────────────────────────
│   │
│   ├─ ★ 회원가입 ·························· LNB 없음, 스텝 흐름
│   │   ├─ Step 1: 약관·개인정보 동의
│   │   ├─ Step 2: PASS 본인인증
│   │   ├─ Step 3: 회원정보 입력
│   │   └─ Step 4: 가입 완료
│   │
│   ├─ ★ 로그인 ···························· LNB 없음
│   ├─ ★ 아이디 찾기 ······················· LNB 없음
│   └─ ★ 비밀번호 찾기·재설정 ·············· LNB 없음
│
├─── 헤더 유틸리티 ────────────────────────────────
│   │
│   ├─ 사이트맵 (≡ 아이콘)
│   │
│   └─ ★ 마이페이지 (로그인 시만) ·········· 좌측 사이드바
│       ├─ 메인 대시보드 (기본 진입)
│       ├─ 데이터 즐겨찾기
│       ├─ 이용 내역 조회
│       │   ├─ 다운로드 내역 (탭)
│       │   ├─ 저작 내역 (탭)
│       │   └─ AI 사용 내역 (탭)
│       └─ 회원 정보 관리
│
└─── 관리자 (/admin, 별도 레이아웃) ───────────────

    ├─ ★ 대시보드 (기본 진입) ··············· 좌측 사이드바

    ├─ 회원 현황 관리
```

### 0-2. 시안 수량 총괄

| 구분 | 시안 | 장수 |
|------|------|------|
| **메인 페이지** (별도 요청서) | 비로그인 모드A + 모드B + 쇼케이스 + 로그인 + 컬러 정리 | ~5장 |
| **GNB 하위** | 학습데이터 + 워크스페이스(로그인/비로그인) + 소식&참여 + 이용안내 | 5장 |
| **회원** | 회원가입 2 + 로그인 1 + 아이디찾기 1 + 비밀번호재설정 1 | 5장 |
| **마이페이지** | LNB 프레임 + 이용내역 탭 | 1장 |
| **관리자** | 관리자 전용 레이아웃 + LNB | 1장 |
| **공통** | LNB 활성/비활성, 브레드크럼 패턴 | 1장 |
| | **합계** | **~18장** |

## 1. 요청 범위

- GNB 클릭 시 진입하는 **각 섹션의 하위 메뉴(LNB) 구조**만 디자인
- 콘텐츠 영역은 **빈 placeholder** (추후 개별 화면 디자인)
- 공통: 헤더(GNB) + 푸터는 메인 페이지 시안 그대로 적용
- 비로그인/로그인 분기가 필요한 메뉴는 명시

---

## 2. 디자인 제약

메인 페이지 요청서(`design-request-main-page.md`)와 동일:

| 항목 | 방침 |
|------|------|
| **디자인 시스템** | KRDS 최소 준수 + 자유 디자인 요소 적용 |
| **금지 라이브러리** | shadcn, Radix UI 사용 금지 |
| **반응형** | 목업에서 제외 (PC만) |
| **애니메이션/모션** | 목업에서 제외 |

---

## 3. 공통 네비게이션 요소

각 섹션 페이지에 공통 적용할 요소:

| 요소 | 설명 |
|------|------|
| **브레드크럼** | 홈 > {GNB 메뉴} > {현재 하위 메뉴} |
| **페이지 타이틀** | 현재 하위 메뉴명 |
| **LNB (좌측 or 상단)** | 하위 메뉴 목록. 현재 활성 메뉴 강조 |
| **콘텐츠 영역** | 빈 placeholder (회색 박스 + "콘텐츠 영역" 텍스트) |

> LNB 배치 방식 (좌측 사이드바 / 상단 탭 / 혼합)은 디자이너 제안 가능.
> 단, **섹션 간 일관성** 유지 (전체 메뉴가 같은 패턴이면 좋음)

## 4. GNB 메뉴별 하위 구조

### 4-1. 학습데이터

> GNB 클릭 시 **검색·목록 페이지로 바로 진입** (별도 LNB 없음)

```
학습데이터
└─ 검색·목록 (기본 진입) ──── 하위 메뉴 없이 단일 페이지 흐름
    └→ 상세 (목록에서 클릭)
        └→ 다운로드 신청 (모달)
```

- LNB 불필요 — 검색→목록→상세 순차 흐름
- 상세 진입 시 브레드크럼: 홈 > 학습데이터 > 데이터셋명
- **시안 1장**: 검색·목록 페이지 (콘텐츠 비움, 검색바+필터 영역만 placeholder)

---

### 4-2. 워크스페이스 (로그인 필수)

> 비로그인 시 **LNB 메뉴 구조는 노출하되, 콘텐츠 영역 블러/잠금 + 로그인 유도 오버레이** 표시

```
워크스페이스
├─ 대시보드         ← 기본 진입
├─ 업로드 영상
├─ 저작도구         ※ MF Remote 임베딩 영역
├─ 내 학습데이터
├─ 데이터 증강
└─ 생성형 AI
```

| 하위 메뉴 | 비고 |
|----------|------|
| **대시보드** | 워크스페이스 기본 진입. **진행중** 작업 허브: 업로드 진행 상태, 저작·증강 작업 현황, 진행중 다운로드 상태·재시도 관리. ※ 마이페이지 대시보드(완료 이력 요약)와 역할 구분 |
| **업로드 영상** | 영상 업로드 폼 + 업로드 현황 목록 |
| **저작도구** | MF Remote 임베딩 — 콘텐츠 영역에 "저작도구(외부 모듈) 로딩 영역" placeholder |
| **내 학습데이터** | 저작 결과 목록 + 상세 + 다운로드 |
| **데이터 증강** | 증강 실행 폼 + 사용량·기여도 |
| **생성형 AI** | 이미지 업로드 + 영상 생성 요청 |

**시안 필요**:
- 로그인 상태: LNB 6개 메뉴 + 빈 콘텐츠 (대시보드 활성)
- 비로그인 상태: LNB 6개 메뉴 노출 + 콘텐츠 영역 블러/잠금 + 로그인 유도 오버레이

---

### 4-3. 소식&참여

```
소식&참여
├─ 데이터 현황       ← 기본 진입
├─ 공지사항
├─ FAQ
├─ 문의하기
└─ 활용사례
```

| 하위 메뉴 | 하위 화면 | 비고 |
|----------|----------|------|
| **데이터 현황** | 단일 페이지 | 시각화 대시보드 |
| **공지사항** | 목록 / 상세 | 목록이 기본 |
| **FAQ** | 목록(카테고리별) / 상세 | 아코디언 형태 가능 |
| **문의하기** | 작성 / 내 문의 목록 / 상세 | 로그인 필수. 내 문의 목록은 로그인 시 노출 |
| **활용사례** | 목록 / 상세 | 카드 레이아웃 |

**시안 필요**:
- LNB 5개 메뉴 + 빈 콘텐츠 (데이터 현황 활성)

---

### 4-4. 이용안내

```
이용안내
├─ 서비스 소개       ← 기본 진입
├─ 이용 가이드
├─ 이용약관
└─ 개인정보처리방침
```

| 하위 메뉴 | 비고 |
|----------|------|
| **서비스 소개** | 정적 콘텐츠 페이지 |
| **이용 가이드** | 단계별 가이드 (검색→다운로드, 저작도구, AI 증강) |
| **이용약관** | 관리자 업로드 문서 표시. 푸터에서도 접근 가능 |
| **개인정보처리방침** | 관리자 업로드 문서 표시. 푸터에서도 접근 가능 |

**시안 필요**:
- LNB 4개 메뉴 + 빈 콘텐츠 (서비스 소개 활성)

---

### 4-5. 회원 (독립 페이지)

> GNB 외부 — 헤더 "로그인"/"회원가입" 버튼 또는 로그인 유도에서 진입하는 독립 흐름.
> 공통 헤더(GNB) + 푸터 적용. LNB 없음.

#### 회원가입 (스텝 흐름)

```
회원가입
├─ Step 1: 약관·개인정보 수집 동의
├─ Step 2: PASS 휴대폰 본인인증
├─ Step 3: 회원정보 입력 (아이디·비밀번호·소속 등)
└─ Step 4: 가입 완료
```

| 단계 | 비고 |
|------|------|
| **약관·개인정보 동의** | 필수/선택 약관 구분, 전체 동의 체크박스, 약관 본문 펼침 |
| **PASS 본인인증** | 외부 팝업/리다이렉트 (EXTSYS-003). 인증 완료 후 자동 복귀 |
| **회원정보 입력** | 아이디 중복확인, 비밀번호 정책 표시, 소속·연락처·이메일 등 |
| **가입 완료** | 완료 안내 + "로그인하기" CTA |

#### 로그인

```
로그인 (단일 페이지)
├→ 아이디 찾기 (링크)
├→ 비밀번호 찾기 (링크)
└→ 회원가입 (링크)
```

| 요소 | 비고 |
|------|------|
| **로그인 폼** | 아이디·비밀번호 입력 + 로그인 버튼 |
| **부가 링크** | 아이디 찾기 · 비밀번호 찾기 · 회원가입 |

#### 아이디 찾기

```
아이디 찾기
├─ Step 1: PASS 휴대폰 본인인증
└─ Step 2: 아이디 안내 (일부 마스킹 표시)
```

#### 비밀번호 찾기·재설정

```
비밀번호 찾기
├─ Step 1: 아이디 입력
├─ Step 2: PASS 휴대폰 본인인증
└─ Step 3: 새 비밀번호 설정 (정책 안내 표시)
```

**시안 필요**:
- 회원가입: 스텝 인디케이터 포함 대표 1장 (Step 1 약관동의 활성) + Step 3 정보입력 1장
- 로그인: 1장
- 아이디 찾기 결과 화면: 1장
- 비밀번호 재설정 화면: 1장

### 4-6. 마이페이지 (헤더에서 진입, 로그인 시만)

> GNB가 아닌 **헤더 유틸리티**에서 진입. 별도 LNB 구조 필요.

```
마이페이지
├─ 메인 (대시보드)    ← 기본 진입
├─ 데이터 즐겨찾기
├─ 이용 내역 조회
│   ├─ 데이터 다운로드 내역
│   ├─ 데이터 저작 내역
│   └─ AI 사용 내역 (증강 + 생성형 AI) / 기여도 상세
└─ 회원 정보 관리
```

| 하위 메뉴 | 비고 |
|----------|------|
| **메인 (대시보드)** | **완료된** 이용 이력 요약 허브: 다운로드 완료 건수, 저작 완료 건수, AI 사용량·기여도 요약 카드. 각 카드 클릭 → 해당 하위 메뉴. ※ 워크스페이스 대시보드(진행중 작업)와 역할 구분 |
| **데이터 즐겨찾기** | 즐겨찾기 데이터셋 리스트 |
| **이용 내역 조회** | **하위 3개 탭** (다운로드 / 저작 / AI 사용) |
| **회원 정보 관리** | 진입 시 비밀번호 재확인 필요 |

**시안 필요**:
- LNB 4개 메뉴 + 빈 콘텐츠 (메인 활성)
- "이용 내역 조회" 선택 시 내부 탭 3개 표시 (LNB 내 depth 2 또는 콘텐츠 영역 상단 탭)

### 4-7. 관리자 (/admin)

> 사용자 포털과 별도 레이아웃. 관리자 전용 GNB 또는 사이드바.

```
관리자
├─ 대시보드 (메인)    ← 기본 진입
├─ 다운로드 현황 관리
├─ 회원 현황 관리
├─ 공지사항 관리
├─ FAQ 관리
├─ 문의 관리
├─ 활용사례 관리
└─ 콘텐츠 관리
```

| 하위 메뉴 | 비고 |
|----------|------|
| **대시보드** | 요약 카드 + 상세 링크 허브 |
| **다운로드 현황 관리** | 통계 차트 + 엑셀 내보내기 |
| **회원 현황 관리** | 회원 목록·검색·필터 + 계정 삭제 + 엑셀 내보내기 |
| **공지사항 관리** | CRUD + 상단고정·공개기간 |
| **FAQ 관리** | CRUD + 카테고리·정렬 |
| **문의 관리** | 접수 목록 + 답변 작성 + 상태 관리 |
| **활용사례 관리** | CRUD |
| **콘텐츠 관리** | 이용약관·개인정보처리방침 WYSIWYG 편집·배포·개정이력 관리 (버전별 시행일 지정) |

**시안 필요**:
- 관리자 LNB (좌측 사이드바 권장 — 메뉴 8개) + 빈 콘텐츠 (대시보드 활성)
- 관리자 레이아웃은 사용자 포털과 다를 수 있음 (디자이너 제안 가능)

## 5. 디자이너 참고사항

### LNB 패턴 제안

| 패턴 | 적합한 곳 | 설명 |
