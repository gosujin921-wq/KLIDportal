Hero Design Concept

KLID
AI 영상학습 사용자 포털

데이터를 찾고, 키우고, 바로 쓸 수 있게 만드는 AI 영상학습 사용자 포털의 브랜드 경험 방향.

Reference Learning

레퍼런스에서 가져올 건
스타일이 아니라 설득 구조입니다

좋은 브랜드 컨셉서는 예쁜 화면을 먼저 보여주지 않습니다. 먼저 서비스가 왜 필요한지 정의하고, 하나의 핵심 단어로 압축한 뒤, 그 단어가 비주얼·모션·서비스 접점으로 어떻게 확장되는지 보여줍니다.

01 Problem

무엇이 헷갈리는가

기존 포털처럼 보이는 순간, 생성·증강·저작이라는 차별성이 흐려집니다.

02 Essence

한 단어로 압축

서비스 가치를 길게 설명하기 전에 모두가 기억할 수 있는 기준어가 필요합니다.

03 System

형태로 반복

큐브, 그리드, 연결선, 모션이 같은 의미를 반복해서 말해야 합니다.

04 Touchpoint

접점으로 확장

히어로에서 끝나는 것이 아니라 검색, 로딩, 빈 상태, 완료 안내까지 이어져야 합니다.

Reframing

이 포털은
다운로드 페이지가 아닙니다

원본 데이터를 탐색하고, AI로 증강·생성하고, 워크스페이스에서 활용 가능한 학습 자산으로 만드는 환경입니다.

Not Only

검색과 반출

데이터를 찾아 내려받는 기능은 기본값입니다. 여기서 끝나면 기존 AI Hub형 포털과 차이가 약합니다.

But Also

증강과 생성

원본이 부족하거나 조건이 맞지 않을 때, AI로 학습데이터의 양과 조건을 확장합니다.

Finally

활용과 관리

프로젝트 단위로 담고, 정리하고, 다시 활용하는 워크스페이스 경험으로 연결합니다.

Brand Essence
GROW

원본 데이터가 끝이 아니라 시작이 되는 포털.
데이터가 학습 가능한 자산으로 자라는 경험을 보여줍니다.

GROW는 연결·증강·생성·활용을 한 번에 묶는 가장 쉬운 기준어입니다. 어려운 영어 조합 대신, 포털이 사용자에게 주는 결과를 바로 말합니다.

Experience Flow

사용자 경험은
Find → Grow → Use로 정리합니다

Find탐색

조건 검색, 데이터 카드, 상세 정보로 필요한 원본 데이터를 발견합니다.

→
Grow성장

AI 증강과 생성으로 부족한 데이터를 늘리고 조건을 다양화합니다.

→
Use활용

워크스페이스에 담아 프로젝트 목적에 맞게 반출·저작·관리합니다.

Visual Principle

시각 언어는
Unit → Cluster → System

하나의 데이터 단위가 묶이고, 확장되고, 정렬되며 하나의 학습 데이터 시스템이 됩니다.

Concept A
A · Main Hero Direction
SYSTEM

데이터 큐브와 그리드 구조를 중심으로, 원본 데이터가 증강·생성을 거쳐 정렬된 학습 자산이 되는 과정을 보여줍니다. 공공기관 포털의 신뢰감과 기술 플랫폼의 확장성을 가장 안정적으로 전달합니다.

전문성신뢰감확장성메인 히어로 적합
Concept B
B · Service Guide Direction
GUIDE

데이터의 최소 단위를 캐릭터화해, 복잡한 탐색·증강·생성·완료 과정을 쉽게 이해시키는 안내자입니다. 귀여움이 목적이 아니라, 어려운 데이터 기능을 낮은 진입장벽으로 전달하는 UX 장치입니다.

Hero Storyboard

히어로는 기능 나열이 아니라
데이터가 자라는 장면이어야 합니다

01

Data Unit

하나의 영상·이미지 세트가 큐브 형태로 등장합니다. 원본 데이터의 시작점을 명확히 보여줍니다.

02

Condition

시간대, 날씨, 이벤트, 라벨 정보가 주변 그리드로 연결됩니다.

03

Growth

AI 증강·생성으로 큐브가 늘어나며 데이터의 조건과 수량이 확장됩니다.

04

Asset

확장된 데이터가 워크스페이스 안에서 정렬되어 학습 자산으로 완성됩니다.

Selection Guide

구분역할적용 위치선택 이유
Concept A · SYSTEM브랜드의 얼굴메인 히어로, 핵심 소개, 제안서 첫 장공공기관 포털에 필요한 전문성·신뢰감·확장성을 가장 강하게 전달합니다.
Concept B · GUIDE사용자 안내자온보딩, 로딩, 빈 상태, 완료/주의 안내복잡한 데이터 과정을 캐릭터 행동으로 낮춰 보여줘 접근성을 보완합니다.
Hybrid가장 안정적인 운영안히어로는 데이터 시스템, 세부 접점은 캐릭터A/B를 억지로 섞지 않고 역할을 분리하면 컨셉이 더 선명해집니다.
