포털은 데이터 마트를 기반으로 학습 데이터셋을 제공하며, 사용자는 데이터를 검색·반출하거나 직접 데이터를 제작할 수 있는 구조로 검토되었다.

---

## 2. 포털 기본 방향

포털은 사용자의 이용 목적을 기준으로 크게 두 가지 관점에서 구성한다.

### 2.1 학습 데이터셋 검색 및 반출

- 사용자가 필요한 학습 데이터셋을 검색한다.
- 검색 결과를 확인한 후 반출 요청을 진행한다.
- 요청된 데이터는 하나의 학습 데이터셋으로 묶어 제공한다.
- 반출 요청 후 데이터 생성 상태와 다운로드 가능 여부를 확인할 수 있어야 한다.

### 2.2 학습 데이터셋 제작

- 사용자가 외부 영상 또는 이미지를 업로드한다.
- 영상은 저작 도구를 통해 라벨링할 수 있다.
- 이미지는 생성형 AI를 통해 영상 생성에 활용할 수 있다.
- 올해 구축 범위에서는 기능을 과도하게 확장하지 않고, 검색·반출·제작이라는 핵심 흐름을 우선 정리한다.

---

## 3. 데이터 마트와 포털의 관계

포털의 데이터는 데이터 마트를 기반으로 구성한다.

- 데이터 마트에서 학습 데이터셋이 포털로 전달된다.
- 내부에서 생성한 학습 데이터셋은 포털 구조에 맞춰 제공한다.
- 외부에서 들어오는 데이터셋은 내부 구조에 맞게 분해하기 어려울 수 있으므로, 통째로 등록하는 방식이 검토되었다.
- 포털에는 AI 모델, 영상·이미지 학습 데이터 등이 등록될 수 있다.
- 데이터 마트는 데이터를 전달하는 역할을 하고, 포털 내부에서는 데이터 관리·삭제·사용자 관리 등 운영 기능이 필요하다.

## 4. 주요 사용자 시나리오

| 구분 | 내용 |
|---|---|
| 데이터 이용 | 원하는 학습 데이터셋을 검색하고 다운로드한다. |
| 데이터 제작 | 외부 영상/이미지를 업로드하여 라벨링 또는 생성형 AI 기능을 사용한다. |

데이터 이용자는 기존에 등록된 학습 데이터셋을 찾는 것이 목적이고, 데이터 제작자는 직접 영상을 올리거나 이미지를 활용하여 새로운 학습 데이터를 만드는 것이 목적이다.

## 5. 데이터 검색 및 반출 구조

검색은 올해 범위에서 AI 자연어 검색이 아닌 **조건 검색 방식**으로 진행한다.

### 5.1 검색 조건

| 조건 | 방향 |
|---|---|
| 기간 | 기본값은 전체 기간 |
| 이벤트 유형 | 주요 검색 조건 |
| 지역 | 주요 검색 조건 |
| 시간대 | 보조 검색 조건 |
| 키워드 | 보조 검색 조건 |

기간과 시간대는 중요도가 낮고, 실제 핵심 검색 조건은 **이벤트 유형과 지역**으로 정리된다.

---

## 6. 검색 결과 제공 단위

검색 결과는 기본적으로 **영상 단위**로 제공한다.

- 포털에서 검색 결과의 기본 단위는 영상이다.
- 영상 안에 검색 조건에 맞는 이미지가 포함되어 있으면, 해당 영상이 검색 결과에 포함된다.
- 영상과 이미지는 학습 데이터셋 관점에서 하나의 세트로 본다.
- 사용자가 필요한 경우 미리보기를 통해 영상 또는 이미지를 확인할 수 있어야 한다.
- 영상 용량 이슈를 고려하여 **영상 제외 옵션**을 둘지 검토한다.

기본 방향은 **영상+이미지 세트 제공**이다.  
개별 이미지 선택까지 허용하면 복잡도가 높아질 수 있으므로, 올해는 전체 단위에서 영상 포함/제외 여부를 선택하는 수준으로 단순화하는 것이 적합하다.

---

## 7. 검색 카드 방식

사용자가 매번 검색 조건을 직접 입력하는 방식보다는, 자주 사용되는 검색 조건을 **카드 형태**로 제공하는 방식이 논의되었다.

검색 카드는 실제 데이터셋을 미리 묶어두는 것이 아니라, **검색 조건을 미리 구성한 카드**이다.

### 예시

| 카드명 | 기본 조건 |
|---|---|
| 산불 데이터 | 이벤트: 산불 / 지역: 전체 / 기간: 전체 |
| 침수 데이터 | 이벤트: 침수 / 지역: 전체 / 기간: 전체 |
| 화재 데이터 | 이벤트: 화재 / 지역: 전체 / 기간: 전체 |

사용자가 카드를 선택하면 해당 조건으로 검색 결과가 조회되고, 이후 지역·기간·시간대 등 조건을 조정할 수 있다.

검색 카드는 너무 많아지면 탐색성이 떨어질 수 있으므로, 이벤트 유형 중심으로 제한된 개수만 제공하는 방향이 적합하다.

---

## 8. 반출 요청 프로세스

사용자는 검색 결과를 확인한 후 **반출 요청**을 진행한다.

반출 요청을 누르면 현재 선택 또는 조회된 조건을 기준으로 데이터셋이 구성된다.  
이후 서버에서 조건에 맞는 영상·이미지·메타데이터를 묶어 하나의 학습 데이터셋 파일로 생성한다.

### 8.1 반출 요청 시 제공 정보

| 항목 | 내용 |
|---|---|
| 이벤트 유형 | 요청 데이터의 이벤트 유형 |
| 기간 | 데이터 수집 기간 |
| 영상 수 | 포함된 영상 개수 |
| 이미지 수 | 포함된 이미지 개수 |
| 데이터셋 개요 | 요청 데이터 구성 요약 |
| 처리 상태 | 생성 중 / 완료 / 다운로드 가능 |
| 다운로드 기한 | 다운로드 가능 유효기간 |

데이터셋 생성에는 시간이 걸릴 수 있으므로, 사용자가 요청 현황과 처리 상태를 확인할 수 있어야 한다.

## 9. 저작 도구 범위

저작 도구는 기존 관제 지원 시스템의 기능을 모두 가져오는 것이 아니라, 포털 목적에 맞게 제한적으로 적용한다.

### 9.1 저작 도구 포함/제외 기능

| 기능 | 반영 방향 |
|---|---|
| 영상 업로드 | 포함 |
| 이미지 컷 기반 작업 | 포함 |
| 수동 라벨링 | 포함 |
| 증강 기능 | 포함 |
| 오토 라벨링 | 제외 |
| VLM 기능 | 제외 |

저작 도구는 프로젝트를 생성하거나 복잡한 AI 분석을 수행하는 도구가 아니라, 사용자가 업로드한 영상을 기반으로 이미지 컷을 확인하고 라벨링하는 정도의 단순한 기능으로 정리한다.

---

## 10. 생성형 AI 범위

생성형 AI는 포털 내 기능으로 포함한다.

### 10.1 주요 방향

- 외부 이미지를 업로드하여 영상을 생성할 수 있다.
- 생성형 AI는 증강 및 영상 생성 기능을 포함한다.
- 생성된 영상은 사용자가 다운로드할 수 있다.
- 생성형 AI 결과물이 저작 도구로 자동 연동되는 구조는 제외한다.
- 사용자가 필요할 경우 결과물을 다운로드한 후 저작 도구에 수동 업로드한다.

생성형 AI와 저작 도구는 별도 모듈 성격으로 보고, 보안 및 연계 복잡도를 고려하여 올해는 자동 연동하지 않는 방향으로 정리한다.

---

## 11. 업로드 데이터 및 보관 정책

포털에서 중요한 운영 정책은 **데이터 유효기간 관리**이다.

사용자가 업로드한 영상, 생성형 AI 결과물, 저작 도구 작업물, 반출 요청 결과물은 계속 보관하지 않고 유효기간을 명시해야 한다.

### 11.1 정책 방향

| 항목 | 정책 방향 |
|---|---|
| 생성형 AI 결과물 | 생성 완료 후 정해진 기간 내 다운로드 |
| 반출 요청 데이터셋 | 생성 완료 후 다운로드 가능 기간 표시 |
| 저작 도구 작업물 | 작업 가능 기간 및 다운로드 기한 표시 |
| 업로드 영상 | 유효기간 이후 삭제 |
| 만료 데이터 | 자동 삭제 또는 관리자 삭제 |
| 개인정보 포함 데이터 | 업로드 금지 안내 필요 |

생성형 AI 결과물이나 사용자가 직접 업로드한 데이터는 장기 보관하지 않는 것이 원칙이다.  
다운로드 기한과 유효기간은 목록 화면에서 명확하게 표시되어야 한다.

---

## 12. 개인정보 및 보안 이슈

개인정보가 포함된 영상이나 이미지가 업로드될 가능성에 대한 논의가 있었다.

다만 개인정보 비식별화, 영상 암호화, 영향평가 대응 등은 올해 사업 범위를 넘어서는 것으로 보고, 내년 고도화 범위에서 검토하는 방향이 제시되었다.

올해 범위에서는 다음 정도로 대응한다.

- 개인정보가 포함된 데이터는 업로드할 수 없다는 안내 문구 제공
- 문제 발생 시 관리자 삭제 기능 제공
- 영향평가 결과에 따라 필요 시 일부 기능 차단 가능
- 비식별화, 암호화 등은 내년 고도화 범위로 검토

---

## 13. 메인 / 랜딩 페이지 방향

초기에는 로그인 후 대시보드에 학습 데이터 통계와 내 작업 현황을 함께 보여주는 방향이 논의되었으나, 후반부에는 메인 화면을 **서비스 소개형 랜딩 페이지**로 구성하는 방향으로 정리되었다.

### 13.1 랜딩 페이지 구성 요소

| 영역 | 내용 |
|---|---|
| 데이터 통계 | 누적 데이터 수, 이벤트별 데이터 수, 월별 증가량 |
| 지역별 현황 | 지도 기반 데이터 분포 표현 |
| 서비스 소개 | 데이터 검색, 생성형 AI, 저작 도구 기능 안내 |
| 공지사항 | 운영 공지 및 안내 |
| 차별화 메시지 | 단순 데이터 허브가 아니라 제작·재가공까지 가능하다는 점 강조 |

로그인 전 사용자도 포털이 어떤 기능을 제공하는지 이해할 수 있어야 하므로, 랜딩 페이지에는 데이터 현황뿐 아니라 생성형 AI와 저작 도구 기능 소개가 포함되어야 한다.

## 14. 워크스페이스와 마이페이지

작업 현황은 “마이페이지”가 아니라 **워크스페이스** 개념으로 분리하는 방향이 논의되었다.

### 14.1 워크스페이스

워크스페이스는 사용자의 작업 공간이다.

포함 항목:

- 반출 요청 현황
- 처리 상태
- 다운로드 가능 여부
- 다운로드 유효기간
- 생성형 AI 작업 현황
- 저작 도구 작업 현황
- 보관 항목
- 업로드 데이터
- 작업 중 데이터

### 14.2 마이페이지

마이페이지는 회원 정보 관리 용도로 별도 제공한다.

포함 항목:

- 회원 정보 수정
- 인증 정보
- 계정 설정

즉, **워크스페이스는 작업 공간**, **마이페이지는 회원 정보 관리 공간**으로 역할을 나누는 것이 적합하다.

## 15. 계정 및 권한 정책

올해 범위에서는 복잡한 조직 계정 구조를 제외하고, 단일 사용자 기준으로 구성한다.

### 15.1 계정 정책

| 항목 | 방향 |
|---|---|
| 기본 단위 | 1유저 = 1워크스페이스 |
| 기업 계정 하위 사용자 | 올해 제외 |
| 워크스페이스 공유 | 제외 |
| 권한 세분화 | 내년 고도화 범위 |
| 회원가입 인증 | 휴대폰 인증 필요 |
| 외국인 가입 | 제한 방향 |

기업 계정, 하위 사용자, 권한 분리, 공유 워크스페이스 등은 내년 고도화 사업에서 검토한다.  
올해는 한 유저가 하나의 독립된 워크스페이스를 사용하는 구조로 정리한다.

| 화면 | 주요 내용 |
|---|---|
| 랜딩 페이지 | 데이터 통계, 지도 기반 현황, 기능 소개, 공지사항 |

## 21. 최종 정리

이번 회의에서는 AI 영상학습 사용자 포털을 **데이터 검색·반출**, **생성형 AI**, **저작 도구**, **워크스페이스** 중심으로 구성하는 방향이 논의되었다.

올해 구축 범위에서는 복잡한 조직 권한, 자동 연동, 개인정보 비식별화, 영상 암호화 등 고도화 기능까지 확장하지 않고, 사용자가 필요한 데이터를 쉽게 찾고 반출 요청하며, 필요한 경우 직접 제작할 수 있는 기본 구조를 우선 구현하는 것으로 정리되었다.

6월 말까지는 전체 포털 구조가 보이는 1차 목업을 준비하고, 7월 초 기업 간담회에서 이를 기반으로 추가 요구사항을 수렴한다.  
이후 7월 말까지 상세 목업과 룩앤필을 정리하고, 8월부터 개발에 착수하는 일정으로 진행한다.
